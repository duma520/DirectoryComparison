## 1. 什么是目录比较工具？

**一句话介绍**：  
这是一个帮助你快速对比两个文件夹内容差异的工具，告诉你哪些文件相同、哪些缺失、哪些多余。

**适用人群**：  
- 普通用户：备份文件时确认是否完整  
- 办公人员：对比项目文档版本  
- 开发者：检查代码库同步情况  
- 设计师：整理素材时去重  
- 运维工程师：同步服务器文件前后校验  

**核心能力**：  
- 扫描两个目录的所有文件和子目录  
- 标记出共同文件、缺失文件、多余文件  
- 支持按文件名、大小、修改时间对比  
- 可过滤、分组、导出结果

## 2. 快速上手（零基础入门）

### 2.1 下载与安装
1. 前往 [项目地址](https://github.com/duma520/DirectoryComparison) 下载最新版本  
2. 解压后双击 `DirectoryComparison.exe` 即可运行（无需安装）

### 2.2 第一次使用
**场景**：你想对比“D:\照片备份”和“E:\照片备份”两个文件夹是否一致。

1. 打开工具  
2. 点击“目录1”右侧的“浏览...”按钮，选择 `D:\照片备份`  
3. 点击“目录2”右侧的“浏览...”按钮，选择 `E:\照片备份`  
4. 点击工具栏上的“开始比较”按钮（或按 F5）  
5. 等待扫描完成，结果会自动显示在列表中  

**结果怎么看？**  
- 绿色背景：两个目录都有的文件  
- 粉色背景：只在目录1中的文件  
- 橙色背景：只在目录2中的文件

## 4. 功能详解（专业视角）

### 4.1 扫描机制
- 使用 `os.walk()` 递归遍历所有子目录  
- 支持符号链接（但不会追踪）  
- 扫描过程中可随时取消  

### 4.2 比较逻辑
- 以**相对路径**为唯一标识  
- 判断文件是否存在（in_dir1 / in_dir2）  
- 记录文件大小和修改时间，但不自动对比内容（可通过右键“比较内容”手动触发）

### 4.3 过滤模式
| 模式 | 说明 |
|------|------|
| 全部文件 | 显示所有项目 |
| 只显示差异 | 显示只在其中一个目录中的项目 |
| 只显示多出的文件 | 显示只在目录1中的项目 |
| 只显示缺少的文件 | 显示只在目录2中的项目 |

### 4.4 显示模式
| 模式 | 说明 |
|------|------|
| 分组显示 | 按共同、仅目录1、仅目录2分组 |
| 平铺显示 | 所有项目按字母顺序排列 |
| 列表显示 | 类似平铺，但更紧凑（待实现） |

### 4.5 右键菜单功能
| 功能 | 说明 |
|------|------|
| 打开文件位置 | 在资源管理器中打开所在文件夹 |
| 复制路径 | 复制相对路径到剪贴板 |
| 复制文件名 | 复制文件名到剪贴板 |
| 比较内容 | 对两个文件进行二进制对比（待实现） |

### 4.6 历史记录
- 自动记录最近使用的20个目录  
- 可从历史记录对话框快速选择  

### 4.7 设置保存
- 自动保存窗口位置、上次目录、过滤模式  
- 使用 QSettings 跨会话保存

## 5. 使用场景与案例

### 5.1 家庭用户：备份照片
**问题**：你手动复制了照片到移动硬盘，担心漏掉某些文件。  
**解决**：对比原文件夹和备份文件夹，粉色文件是漏掉的，橙色是多出来的（可能是旧照片）。  

### 5.2 办公人员：版本管理
**问题**：项目文档在两个同事电脑上修改后，需要合并。  
**解决**：对比两个文件夹，橙色文件是同事新增的，粉色是你独有的，绿色是共同部分。  

### 5.3 开发者：同步代码库
**问题**：你在两台电脑上开发，有时忘记同步某些文件。  
**解决**：对比两个项目目录，快速找出缺失的文件并手动复制。  

### 5.4 设计师：整理素材
**问题**：下载了多套图标库，很多重复。  
**解决**：对比不同图标库目录，找出重复文件进行去重。  

### 5.5 运维：同步服务器文件
**问题**：使用 rsync 同步后，想确认一致性。  
**解决**：下载到本地后对比源和目标目录，校验完整性。

## 6. 高级技巧与定制

### 6.1 自定义颜色
你可以修改代码开头的颜色定义，例如：
```python
COLOR_EXIST = QColor('#B5EAD7')   # 薄荷绿
COLOR_MISSING = QColor('#FFDAC1') # 蜜桃橙
COLOR_EXTRA = QColor('#FFB7CE')   # 樱花粉
```

### 6.2 添加新过滤模式
在 `CompareMode` 枚举中添加新类型，并在 `filter_results` 中添加对应逻辑。

### 6.3 添加文件内容比较
目前仅对比元数据，如需内容比较，可在右键菜单中添加调用 `filecmp.cmp()` 或哈希对比（如 MD5）。

### 6.4 导出结果为 CSV
可扩展“文件”菜单，将当前结果导出为 CSV 文件，便于在 Excel 中处理。

### 6.5 多线程优化
扫描和比较已使用 QThread，避免界面卡顿。如需更高性能，可考虑批量扫描或缓存。

## 7. 常见问题解答

### Q1: 为什么扫描很慢？
A: 如果目录包含大量文件（如数万张照片），扫描可能需要一些时间。建议耐心等待，或先缩小扫描范围。

### Q2: 可以对比网络路径吗？
A: 可以，直接输入网络路径（如 `\\server\share\folder`）即可，但速度取决于网络质量。

### Q3: 为什么有些文件显示为“-”？
A: 表示该文件在对应目录中不存在，或无法读取元数据（如权限不足）。

### Q4: 如何比较文件内容是否一致？
A: 右键点击文件，选择“比较内容”（需在后续版本中实现）。目前可通过文件大小和修改时间初步判断。

### Q5: 软件会删除或修改我的文件吗？
A: 不会。本工具**只读**，不会对文件进行任何修改。

### Q6: 支持 Linux 或 macOS 吗？
A: 目前为 Windows 版，基于 PySide6 开发，理论上可移植，但未测试。
